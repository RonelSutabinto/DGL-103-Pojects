Debugging example

This webpage is deliberately written with some built-in errors for you to explore. It might look ok-ish in the browser you are using, but it might choke on a browser that is less tolerant of coding mistakes.

Characteristics of a good quality code are efficiency, reliability, robustness, portability, maintainability and readability.

Instructions

  1. Look at the source code in VS Code. Can you identify the invalid HTML and CSS? Correct all the errors you can find and make sure you document your corrections by adding comments. You can use the VSCode extension HTMLHint and your browser's developer tools to help you.
    • Some of the errors could be unclosed elements or badly nested elements. Nesting elements properly is important for code to behave correctly.
    • Some of the code might be valid but it might not make the best use of semantic HTML.
    • Some of the code might be poorly formatted with errors such as incorrect indentation or capitalization.
    • Make sure you know what every line of code does. There may be occasions when you could code the solution in a couple of different ways. Do what you think will make the code as concise and clean as possible.
    • Refer to the Coding Best Practices checklist as you work. Make sure files are well organized and your code is well formatted with consistent indents and spacing.
  2. Run this page through the W3C HTML Validator. Did you find the same mistakes that the Validator found? Correct any problems found by the Validator, then save the web page, and retest until the web page passes the validity test. Take a screenshot of the results.
